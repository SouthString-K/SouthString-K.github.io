# 浙江省机器人大赛：相机标定

这一篇主要整理视觉系统中非常基础但也非常重要的一部分内容，也就是相机标定。

如果不先完成标定，后面的测距、定位、姿态估计都会受到影响。很多时候视觉效果不好，并不是模型本身有问题，而是前面的成像几何没有处理好。

## 相机标定到底在做什么

简单来说，相机标定的目标是求出：

- 相机内参矩阵
- 畸变参数
- 在某些任务里还会求外参

其中最核心的是内参矩阵。

## 针孔相机模型

在理想情况下，三维点 `P = (X, Y, Z)` 投影到图像平面上的关系可以写成：

```text
[u v 1]^T = K [X/Z  Y/Z  1]^T
```

其中 `K` 就是相机内参矩阵：

```text
K = [ fx   0   cx ]
    [  0  fy   cy ]
    [  0   0    1 ]
```

这里：

- `fx, fy` 表示在像素坐标系下的焦距
- `(cx, cy)` 表示主点坐标，也就是光轴和图像平面的交点

## 畸变模型

真实相机还存在镜头畸变，常见的是：

- 径向畸变
- 切向畸变

径向畸变常见写法为：

```text
x_distorted = x(1 + k1*r^2 + k2*r^4 + k3*r^6)
y_distorted = y(1 + k1*r^2 + k2*r^4 + k3*r^6)
```

其中：

```text
r^2 = x^2 + y^2
```

切向畸变通常与 `p1, p2` 两个参数有关。

在 OpenCV 的标定结果中，通常会得到：

- 一个相机内参矩阵 `cameraMatrix`
- 一组畸变参数 `distCoeffs`

## 标定板的作用

标定板本质上是一个几何尺寸已知的参考物体。最常见的是棋盘格标定板。

因为棋盘格每个角点的位置关系是已知的，所以我们可以建立：

- 世界坐标系中的角点位置
- 图像坐标系中的角点位置

然后通过多组图像去拟合出相机参数。

## 标定流程

下面是一个常见且实用的标定流程。

### 1. 准备标定板

一般使用棋盘格标定板，比如：

- `9 x 6` 内角点
- 方格边长 `20 mm` 或 `25 mm`

这里要特别注意：

- OpenCV 里填写的是“内角点数量”
- 不是格子数

### 2. 拍摄标定图片

用待标定的相机从不同角度拍摄标定板图片，建议：

- 多拍一些，通常 `15` 到 `25` 张较合适
- 尽量覆盖不同位置、不同角度、不同距离
- 让棋盘格出现在画面的中心、边缘、倾斜状态

这样能让标定结果更稳定。

### 3. 提取角点

在 OpenCV 中，一般使用：

```python
cv2.findChessboardCorners()
```

来寻找棋盘格角点，再用：

```python
cv2.cornerSubPix()
```

进行亚像素级优化。

### 4. 构建世界坐标点

假设标定板为 `9 x 6` 内角点，方格边长为 `square_size`，那么世界坐标通常按如下方式构建：

```python
objp = np.zeros((6 * 9, 3), np.float32)
objp[:, :2] = np.mgrid[0:9, 0:6].T.reshape(-1, 2)
objp *= square_size
```

这里得到的是每个角点在标定板平面中的三维坐标，一般默认 `Z = 0`。

### 5. 调用标定函数

OpenCV 中最常用的是：

```python
ret, cameraMatrix, distCoeffs, rvecs, tvecs = cv2.calibrateCamera(
    objpoints,
    imgpoints,
    gray.shape[::-1],
    None,
    None
)
```

输出中：

- `cameraMatrix` 是内参矩阵
- `distCoeffs` 是畸变参数
- `rvecs` 和 `tvecs` 是每张图对应的外参

## 如何获得内参矩阵

标定完成后，OpenCV 会直接输出内参矩阵，例如：

```python
print(cameraMatrix)
```

可能得到类似结果：

```python
[[921.45   0.   640.12]
 [  0.   919.87 362.54]
 [  0.     0.     1.  ]]
```

这就表示：

- `fx = 921.45`
- `fy = 919.87`
- `cx = 640.12`
- `cy = 362.54`

这四个参数共同描述了相机从三维到二维投影时的内部几何关系。

## 标定质量如何判断

常见的判断方式是看重投影误差。

重投影误差的基本思想是：

1. 用求出来的相机参数，把世界坐标点重新投影回图像
2. 看投影点和真实检测到的图像点差多少

常见误差公式可以写成：

```text
e = (1/N) * Σ || p_i - p_i_hat ||
```

其中：

- `p_i` 是真实检测到的图像点
- `p_i_hat` 是根据标定结果重新投影得到的点

误差越小，通常说明标定结果越好。

## 去畸变

得到内参与畸变参数后，就可以对图像做去畸变处理。

OpenCV 常用方法：

```python
undistorted = cv2.undistort(img, cameraMatrix, distCoeffs)
```

这样得到的图像会比原图更接近理想成像模型。

## 一个比较完整的思路

如果要自己做一次完整标定，大致可以按这个顺序：

1. 准备棋盘格标定板
2. 拍摄多张不同角度图片
3. 提取角点
4. 构建世界坐标点
5. 调用 `cv2.calibrateCamera`
6. 保存 `cameraMatrix` 和 `distCoeffs`
7. 用去畸变图像检查效果

## 为什么这一步很重要

如果后续你要做：

- 测距
- 双目视觉
- 位姿估计
- 目标定位
- 机械臂抓取

那么相机标定基本是绕不开的。很多视觉任务后面效果不好，并不是模型本身有问题，而是前面的成像几何没有处理好。

## 最后

相机标定解决的是“视觉结果是否可信”的问题。

在机器人大赛里，很多人会把精力都放在模型本身，但真正比赛时，往往是这些基础问题决定系统能不能稳定工作。
